# PHASE 1 — FOUNDATION
## Real Backend, Database & Hosting

**Goal:** Replace GitHub Pages with a real server. Set up a database. Move from static demo data to a real API layer.

**Time estimate:** 1–2 days with AI assistance

---

### Step 1.1 — Choose Your Stack

Use this exact stack (free tiers available for all):

| Layer | Tool | Why |
|-------|------|-----|
| Frontend hosting | **Vercel** | Free, instant deploys from GitHub |

### Step 1.2 — Set Up Supabase

1. Go to [supabase.com](https://supabase.com) → Create account → New project
2. Name it `eventpulse`, choose a region close to Toronto (US East)
3. Save your **Project URL** and **anon public key** — you'll need these
4. Go to **SQL Editor** and run this schema:



-- VENUES
CREATE TABLE venues (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  address TEXT,
  city TEXT DEFAULT 'Toronto',
  region TEXT DEFAULT 'ON',
  country TEXT DEFAULT 'CA',
  latitude DECIMAL(10, 8),
  longitude DECIMAL(11, 8),
  capacity INTEGER,
  image_url TEXT,
  website TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- EVENTS
CREATE TABLE events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source TEXT NOT NULL,
  source_id TEXT,
  title TEXT NOT NULL,
  description TEXT,
  category TEXT,
  subcategory TEXT,
  event_date DATE NOT NULL,
  start_time TIME,
  end_time TIME,
  venue_id UUID REFERENCES venues(id),
  price_min DECIMAL(10,2),
  price_max DECIMAL(10,2),
  currency TEXT DEFAULT 'CAD',
  ticket_url TEXT,
  image_url TEXT,
  popularity_score INTEGER DEFAULT 0,
  is_verified BOOLEAN DEFAULT false,
  submitted_by UUID,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(source, source_id)
);

-- USERS (extends Supabase auth.users)
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  username TEXT UNIQUE,
  display_name TEXT,
  avatar_url TEXT,
  city TEXT DEFAULT 'Toronto',
  radius_km INTEGER DEFAULT 80,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- SAVED EVENTS
CREATE TABLE saved_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  event_id UUID REFERENCES events(id) ON DELETE CASCADE,
  notify BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, event_id)
);

-- FOLLOWED ARTISTS/VENUES
CREATE TABLE follows (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  entity_type TEXT CHECK (entity_type IN ('artist', 'venue')),
  entity_name TEXT NOT NULL,
  entity_id TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, entity_type, entity_name)
);

-- INDEXES
CREATE INDEX idx_events_date ON events(event_date);
CREATE INDEX idx_events_category ON events(category);
CREATE INDEX idx_events_venue ON events(venue_id);
CREATE INDEX idx_venues_city ON venues(city);
CREATE INDEX idx_saved_user ON saved_events(user_id);
CREATE INDEX idx_follows_user ON follows(user_id);

# PHASE 2 — REAL APIs
## Live Event Data

**Goal:** Pull real events from Ticketmaster, Eventbrite, and Bandsintown into your database on a schedule.

**Time estimate:** 2–3 days

---

### Step 2.1 — Get Your API Keys

#### Ticketmaster (easiest, most data)
1. Go to [developer.ticketmaster.com](https://developer.ticketmaster.com)
2. Sign up → Create App → Copy your **Consumer Key**
3. Free tier: 5,000 calls/day — more than enough to start
4. GTA query example:
   

   https://app.ticketmaster.com/discovery/v2/events.json
     ?apikey=YOUR_KEY
     &city=Toronto
     &countryCode=CA
     &radius=80
     &unit=km
     &size=200
   


#### Eventbrite
1. Go to [eventbrite.com/platform](https://www.eventbrite.com/platform)
2. Sign up as developer → Create a Private Token
3. Endpoint: `https://www.eventbriteapi.com/v3/events/search/`
4. Params: `location.address=Toronto,ON`, `location.within=80km`

#### Bandsintown
1. Go to [artists.bandsintown.com/api](https://artists.bandsintown.com/api)
2. Apply for API access (usually approved within 24 hours)
3. Best for: concert-specific data, artist following

#### Songkick
1. Go to [songkick.com/developer](https://www.songkick.com/developer)
2. Fill out API access form
3. Best for: metro area coverage, venue data

---

### Step 2.2 — Build the Sync Service

Create `/server/services/sync.js`. This runs on a schedule and fetches events from all APIs.

**AI Prompt to use:**
> "Write a Node.js module that fetches events from the Ticketmaster Discovery API for Toronto, CA within 80km radius. Normalize each event to this schema: `{ source, sourceId, title, category, eventDate, startTime, venueName, venueAddress, latitude, longitude, priceMin, priceMax, ticketUrl, imageUrl }`. Handle pagination, rate limiting with exponential backoff, and return an array of normalized events."

**Key normalization logic to implement:**



// Category mapping from Ticketmaster segments
const CATEGORY_MAP = {
  'Music': 'Concerts',
  'Sports': 'Sports',
  'Arts & Theatre': 'Theatre',
  'Film': 'Other',
  'Miscellaneous': 'Other',
  'Comedy': 'Comedy',
  'Family': 'Festivals',
};

// Deduplication: before inserting, check for existing event
// Match on: same venue + same date + title similarity > 80%
async function isDuplicate(event, supabase) {
  const { data } = await supabase
    .from('events')
    .select('id, title')
    .eq('event_date', event.eventDate)
    .eq('venue_id', event.venueId);
  
  if (!data?.length) return false;
  
  return data.some(existing => 
    stringSimilarity(existing.title, event.title) > 0.8
  );
}

### Step 2.3 — Schedule the Sync

Add to your server's `index.js`:



import cron from 'node-cron';
import { syncAllSources } from './services/sync.js';

// Run every 6 hours
cron.schedule('0 */6 * * *', async () => {
  console.log('Starting event sync...');
  await syncAllSources();
  console.log('Sync complete');
});

// Also expose a manual trigger endpoint (protect with API key)
app.post('/api/admin/sync', adminAuthMiddleware, async (req, res) => {
  await syncAllSources();
  res.json({ success: true });
});

# 🤖 AI ASSISTANT TIPS

When working with Claude, Cursor, or Copilot on this project, use these strategies:

### Always give context first
Start prompts with:
> "I'm building EventPulse, a React/Vite event aggregation calendar app. The backend is Node.js/Express on Railway, database is Supabase PostgreSQL. Here is the current state of [file]..."

### Request specific file output
> "Write the complete contents of `/server/services/ticketmaster.js`"

### Ask for error handling explicitly
> "...and include try/catch, log errors to console, and return null on failure instead of throwing"

### Break large features into steps
Instead of "build the whole auth system", ask:
1. "Add the Supabase client to the frontend"
2. "Create the AuthModal component"
3. "Add the login handler"
4. "Persist the session in Zustand"
